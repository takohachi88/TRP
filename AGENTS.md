# Codex Project Instructions

このリポジトリでCodexが作業するときの基本ルールです。

## Project Overview

- このプロジェクトは、UnityでTRPという独自のレンダーパイプラインを制作するプロジェクトです。
- `unity-cli-loop`（https://github.com/hatayama/unity-cli-loop/tree/v3-beta）を導入済みであり、Unityの操作はこれを通じて行えます。

## Git

- コミット、ブランチ作成、マージ、リベース、タグ作成など、履歴や作業ツリーを変更するGit操作は原則として行わないでください。
- Git操作が必要に見える場合も、まずメンバーに確認してください。
- 状況確認のための読み取り専用Gitコマンドは、必要に応じて使用してかまいません。

## Work Planning

- 実装やファイル変更を伴う作業で計画を立てた場合は、作業開始前にメンバーの承認を取ってください。
- 小さな調査や読み取り専用レビューは、必要に応じて先に進めてかまいません。
- 承認後も、計画から外れる変更が必要になった場合は再確認してください。

## Unity Assets

- シーン（`.unity`）やPrefab（`.prefab`）は原則として変更しないでください。
- 変更が不可避な場合は、理由と変更範囲を説明して事前承認を取ってください。
- ScriptableObjectなどのアセット変更も、コード変更だけで完結できない場合に限定し、事前に相談してください。

## Main Working Area

- 主な作業対象は `Packages/tako.trp/` です。
- 既存の命名に合わせて、小さく安全に変更してください。

## Coding Style

- コーディングにあたっては、日本語のコメントを十分に記載してください。
- シェーダーのコーディングを行う際、色の計算など精度が重要でない場合、型はfloatでなくhalfを使用し、定数でない除算はrcp関数を使うようにしてください。
- ランタイムで毎フレーム実行する処理ではGCAllocが発生しないようにしてください。

## Communication

- 重要な判断、前提、リスクは日本語で簡潔に共有してください。
- 仕様が曖昧な場合は、無理に決め打ちせず確認してください。
